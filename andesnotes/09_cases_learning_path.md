# Case Learning Path

Updated: 2026-03-08 01:44 (Asia/Shanghai)

## Validation Boundary
Only records verifiable local observations and executable paths.

## Recommended Learning Order
### Path A — Built-in ANDES cases
Goal: verify engine and CLI pipeline
1. Environment setup
2. CLI self-check
3. Baseline power-flow run

### Path B — Dynamic and control-focused cases
Goal: verify disturbance and response workflows

### Path C — Notebook/demo workflows
Goal: transfer from CLI validation to notebook orchestration

## Guideline
Use small, deterministic cases first; scale up only after reproducible baseline success.
